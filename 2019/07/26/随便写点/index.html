想法记录

  1. 组件 -> 区块 -> 模板，这是我们所做事情的正常化节奏。
  2. 解决问题的思路都是通的，之前是单点解决问题，后来是线解决问题，再后来是面解决问题。
  3. 网络这个蛋糕我一定要啃下来，感觉网络的世界很有意思。- 主啃的一个点。 想从这个单点出一个系列的文章。
  4. 自己如果比较懒的话，那就搞自己的脚手架。
  5. 目前的问题是想了很多，但感觉做的时候会比较少，我应该需要聚焦一些主题，然后产出文章，接着下一个。我目前大概有20+个主题，需要一个一个来，跟估时一样，要有自己的节奏。
  6. 自己累的时候，直接早点回家休息，休息一周就会调节回来，希望自我调节可以做的好点，判断标准 是否找到了自己。
  7. 视频跟文档都是学习，没有区别。
  8. 多记录，语雀和xmind都可以。
  9. 主要学习的东西: react shell mysql socket ts grahql 目前把这六个方向作为我的主要方向以及辅助PPT。
  10. do not be stupid
  11. 主要了解的方向思路: 网关服务 搭建服务化 serverless schema2**(配置化)(以及围绕产生schema所形成的方式 搭建 流程图 表单 手写 自定义)。
  12. 想让自己变得真正的强大，包括技术知识的方面以及业务推动的方面。 业务上: 发现 -> 决策 -> 方案 -> 结果这条路线是没有问题的。
  13. 多找人交谈一些想法，如果有些事情要做就做，不要把自己逼到角落，不要拖延。
  14. 技术伴随业务成长，找到自己的路，在业务中成长，但也不要沉浸在业务里头，谨记。是否发现业务里的问题，然后加以解决，并且拿到数据，这是最好的方式。
  15. 没必要为很多很多的知识以及不解感到沮丧，因为那是别人花费很多精力来做完的，你想在很短时间内了解清楚，本身也是种妄想。
  16. 没事多看书，看书！看书！看书！
  17. 善于总结
  18. 看源码 -> debug!debug!debug!
